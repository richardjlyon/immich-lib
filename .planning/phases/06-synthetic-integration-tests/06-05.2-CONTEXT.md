# Context: Unique Base Images for Test Fixtures

## Problem Discovered

During 06-05-02 execution, integration tests revealed that Immich's CLIP model groups fixtures from the same base image together, regardless of scenario.

**Example:** C1, C2, C3, C4, C5, C6 all got grouped into ONE duplicate group because they all use `base_food.jpg` or `base_animal.jpg`.

This makes it **impossible to test scenarios in isolation**.

## Root Cause

`src/testing/fixtures.rs` reuses ~8 base images across 34 scenarios:
- `base_landscape.jpg` - used by W1, W2, W3, X9, X10, X11
- `base_urban.jpg` - used by W4, W5, W6
- `base_portrait.jpg` - used by W7, W8
- `base_food.jpg` - used by C1, C2, C3
- `base_animal.jpg` - used by C4, C5, C6
- `base_night.jpg` - used by C7, C8
- `base_macro.jpg` - used by F1, F2, F3
- `base_abstract.jpg` - used by F4, F5, F6
- `base_indoor.jpg` - used by F7, X1, X2, X4
- `base_outdoor.jpg` - used by X3, X5, X6, X7, X8

## Solution Required

1. **Source 34 unique Unsplash photos** - one per scenario
2. **Update `fixtures.rs`** - each scenario uses its own base image
3. **Regenerate fixtures** - `./target/debug/immich-dupes generate-fixtures`
4. **Verify isolation** - re-run integration tests, confirm each scenario = one group

## Key Files

- `src/testing/fixtures.rs` - defines which base image each scenario uses
- `tests/fixtures/base/` - where base images are stored
- `tests/fixtures/*/` - generated scenario fixtures

## Estimated Effort

~15-20 minutes:
- Find 34 Unsplash URLs (can use Unsplash API or manual selection)
- Update fixtures.rs with unique base image names per scenario
- Regenerate and verify
